Parallelprogrammierung – Multiprocessing
Programmierparadigmen

Unix pipe

Ein erstes Beispiel für Multiprocessing: der Kommandozeilenoperator |

  • Eingabe in der Unix-Kommandozeile: 

    cat file.txt | sort
    • cat wird in einem und sort in einem zweiten Prozess ausgeführt.
    • Der sort-Prozess wartet auf alle Bytes des cat-Prozesses bevor er die Sortierung beginnt.
    • Wenn die Maschine zwei Prozessoren besitzt, können die beiden Prozesse auf diese verteilt werden.
    • Der Pipe-Operator führt automatisch Multiprocessing durch und sorgt für die Inter-Prozess-Kommunikation (IPC).

  • Aufbau der Pipe: 

                           pipe
               +--------|-------+
    writer---->|WriteEnd|ReadEnd|<----reader
               +--------+-------+
  • Es gibt zwei grundsätzliche Wege im Programmcode, zwei Prozesse zu erzeugen:
    1. Ein Eltern-Prozess „forks” (clones, spawns, spaltet ab) einen Kind-Prozess und beide Prozesse arbeiten Code aus demselben Programm ab.
      • Eine Fallunterscheidung (if-else o. ä.) trennt den Eltern-Code vom Kind-Code.
    2. Der Eltern-Prozess spaltet einen Kind-Prozess ab, der dann von einem eigenen Programm ausgeführt wird.

  • Ein zweites Beispiel: 

    sleep 5 | echo 'Hello, world!'
    • sleep sendet keine Bytes in die Pipe und echo erwartet keine.
    • Hello, world! erscheint auf dem Bildschirm und 5 Sekunden später wird die Eingabeaufforderung wieder angezeigt.

Ein Beispiel für Multiprocessing in C

Überblick

  • When das Programm pipeExample.c übersetzt und dann von der Kommandozeile gestartet wird, wird es in einem Prozess, dem Elternprozess ausgeführt.

  • Kommando zur Ausführung 

    % ./pipeExample   ## unter Windows: % pipeExample
  • Der Eltern-Prozess ruft die Bibliotheksfunktion fork() auf, um einen Kind-Prozess abzuspalten.
    • fork() macht zu diesem Zweck einen Systemaufruf.
    • Mögliche Rückgabewerte von fork()
      • -1 Es ist ein Fehler aufgetreten. Aus welchen Gründen auch immer konnte kein weiterer Prozess erzeugt werden.
      • 0 wird im Erfolgsfall an den Kind-Prozess gegeben
      • > 0 wird im Erfolgsfall an den Eltern-Prozess gegeben; konkret wird die process id (pid) des Kind-Prozesses geliefert, die immer eine positive ganze Zahl ist.
  • Der Eltern- und der Kind-Prozess führen beide den Code aus, der dem Aufruf von fork() folgt.

    • Eine Fallunterscheidung (if-else o. ä.) stellt sicher, dass der Eltern-Prozess etwas anderes als der Kind-Prozess tut: 

      pid_t pid = fork();   /** pid_t is a signed integer data type **/
if (-1 == pid) {      /** error: couldn't clone a new process **/
    ... }   
else if (0 == pid) {  /** child code follows **/
    ... }
else {                /** parent code follows **/
    ...}
  • Das Elter wartet auf die Beendigung des Kinds, so dass das Kind nicht zur Waise („zombie“ wird.
    • Wenn das Kind terminiert, unterrichtet das Betriebssystem das Elter.
    • Falls das Elter schon beendet ist und nicht mehr benachrichtigt werden kann, wird das Kind zu einer Waise. Der Prozess existiert noch ist aber tot.

Starten des Programms pipeExample.c in der Unix-shell

bash-3.2$ ls pipeExample.c 
pipeExample.c
bash-3.2$ gcc -o pipeExample pipeExample.c 
bash-3.2$ ./pipeExample
